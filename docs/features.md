# FEATURES.md

## 1. 概要

本プロジェクトは、2D グリッド上のキャラクターに対して「命令列（プログラム）」を組み立て、**すべてのコインを取得するとクリア**となるパズルゲームを SvelteKit で実装する。

- プレイヤーはマップ（レベル）ごとに用意された **関数（例: `main`, `F1`, `F2` …）** に命令を配置できる
- 命令には **色条件**を付与でき、タイル上の色と一致しない命令は実行されない（スキップ）
- `Go` で実行開始、`STOP` で **キャラクター状態のみ**リセット（命令は保持）

## 2. 用語

- **マップ/レベル**: タイル（グリッド）の集合。開始位置、壁、コイン、色タイルなどを含む
- **タイル色（tileColor）**: タイルに設定された色（例: `none`, `red`, `blue`, `green`…）
- **命令色（cmdColor）**: 命令に設定できる色（例: `none` または具体色）
- **命令**: 移動・回転・関数呼び出しなど
- **実行エンジン**: 命令列を 1 ステップずつ評価し、キャラクター状態を更新する

## 3. ゲームルール

- 目的: **コインをすべて取得**したらクリア
- キャラクターが **マップ外に出たらリセット**
- 命令を最後まで実行してもコインを取り切れなかった場合:
  - 「プログラムが実行されました」と表示する（失敗扱い）
- `STOP` ボタン:
  - 実行を停止する
  - キャラクターの位置・向き・取得済みコイン等の“実行状態”を初期化する
  - **命令（プログラム編集内容）は保持する**
- `Go` ボタン:
  - 現在の命令を使ってキャラクターが行動を開始する
- 関数呼び出しは再帰・無限ループ可能（ただし安全対策として実行ステップ上限を設ける）

## 4. プレイヤーができること（プログラミング）

### 4.1 関数（レベルごとに可変）

> **重要**: 関数の種類・数・開始点はレベル JSON で決まる。

- 関数の例: `main`, `F1`, `F2`, `F3` …（将来拡張可能）
- レベルによって:
  - `main` がある場合もあれば、**`main` が無く `F1`/`F2` のみ**のレベルもある
  - あるいは `main` と `F1` だけ等の構成もあり得る
- **各関数に配置できる命令数（スロット数）**はレベルごとに異なる

#### 4.1.1 エントリポイント（実行開始関数）

- 実行開始点は `program.entry` で指定する
- `Go` を押したら `entry` 関数から実行を開始する

### 4.2 命令の種類（最小セット）

実装は拡張可能にしつつ、まずは以下をコアとして扱う：

- `MOVE_FORWARD`（前進）
- `TURN_LEFT`（左回転）
- `TURN_RIGHT`（右回転）
- `CALL`（任意の関数呼び出し）
  - UI では `F1`, `F2` のように個別ボタン表示

> `CALL` の呼び出し先（呼べる関数）はレベルの関数定義に従う。

### 4.3 命令の色条件（重要）

- 命令は `cmdColor` を持つ
  - `none`（無条件で実行）
  - `red` / `blue` / `green` …（タイル色一致が必要）
- 命令評価時:
  - 現在タイルの `tileColor` が `X` の場合:
    - `cmdColor` が `X` の命令だけ実行
    - 一致しない命令は **実行されずスキップ**
- `tileColor = none` の扱いはレベル JSON の `capabilities.colorRule` で方針を選べる
  - `matchTileColor`: `cmdColor=none` のみ実行（デフォルト）
  - `allowAllOnNone`: タイル色なしなら全色命令を実行可能（互換用）

## 5. 実行モデル（エンジン仕様）

### 5.1 逐次ステップ実行

- エンジンは 1 ステップごとに「次の命令」を評価し、状態を更新する

### 5.2 呼び出しスタック

- 関数呼び出しは **スタック**で管理する
- 各フレームは `(functionId, instructionPointer)` を持つ
- 命令 `CALL(target)` は新しいフレームを push

### 5.3 無限ループ / 再帰

- 無限ループ・再帰は **制限せず許容**する
- 実行はユーザー操作の `STOP` によって中断できる
- そのため、エンジンは「停止要求」をいつでも受け取れる設計にする（例: 1ステップごとに stop フラグを確認）

### 5.4 マップ外・壁・コイン

- マップ外に出た場合: 即リセット（ルール通り）
- 壁（通行不可タイル）に前進した場合:
  - 既定: その場に留まる（命令は消費）
  - 別挙動もレベル JSON で選択可能
- コイン:
  - タイルに `coin: true` の場合、到達時に取得（消える）
  - すべて取得でクリア

## 6. UI / UX

### 6.1 画面構成

- マップ表示（グリッド・キャラ・コイン・色）
- プログラム編集
  - レベルで定義された関数一覧（例: `F1`, `F2` だけ / `main`, `F1`, `F2` など）
  - 各関数のスロット数もレベル定義に従う
- 命令パレット（前進・回転・関数呼び出し）
  - 関数呼び出しボタンは、レベルで定義された関数に応じて動的に表示
- `Go` / `STOP`、メッセージ表示、速度（任意）

### 6.2 操作

- 命令追加: パレットからスロットへ追加
- 命令削除: スロット上の削除ボタン / ドラッグアウト
- 命令並べ替え: ドラッグで順序変更（任意）
- `Go`: `program.entry` から実行開始
- `STOP`: 実行停止 + 状態リセット（命令は保持）

## 7. JSON でのデータ駆動（最重要）

> 「マップの定義」と「ユーザーができること（関数構成、命令種別、色など）」は **レベル JSON** で定義する。

### 7.1 レベル JSON 構成（案）

- `src/lib/levels/<levelId>.json` のようにレベルごとに配置
- ゲーム起動時に JSON を読み込み、UI とエンジンを構成する

#### 例: レベル定義（関数が可変・main が無いケースにも対応）

```json
{
  "id": "level4",
  "title": "Level 4",

  "grid": {
    "width": 8,
    "height": 8,
    "tiles": [
      { "x": 0, "y": 0, "type": "floor", "tileColor": "none", "coin": false },
      { "x": 1, "y": 0, "type": "floor", "tileColor": "blue", "coin": true }
    ]
  },

  "start": { "x": 0, "y": 0, "dir": "E" },

  "rules": {
    "onOutOfBounds": "reset",
    "onWallCollision": "stay",
  },

  "program": {
    "entry": "F1",
    "functions": {
      "F1": { "maxSlots": 8 },
      "F2": { "maxSlots": 2 }
    }
  },

  "capabilities": {
    "availableCommands": ["MOVE_FORWARD", "TURN_LEFT", "TURN_RIGHT", "CALL"],
    "callTargets": ["F1", "F2"],
    "availableColors": ["none", "red", "blue", "green"],
    "colorRule": "matchTileColor"
  },

  "strings": {
    "success": "クリア！",
    "failExecuted": "プログラムが実行されました",
  }
}
```

- `program.entry` が `main` である必要はない
- `program.functions` のキーがそのまま UI の関数タブ/行になる
- `maxSlots` が各関数の命令数上限
- `capabilities.callTargets` により「呼び出し命令で選べる関数」を制限できる

### 7.2 JSON バリデーション

- Zod 等でスキーマ検証（起動時に fail-fast）
- 不正 JSON の場合は開発時に分かるよう、エラー画面に詳細表示

## 8. 受け入れ基準（Acceptance Criteria）

- レベル JSON を差し替えるだけで
  - **関数の種類（main 有無含む）**
  - 各関数の **スロット数**
  - 使える命令・色・ルール
  が変わる
- `Go` を押すと `program.entry` から実行される
- 命令色とタイル色が一致しない命令は実行されない（スキップ）
- `STOP` でキャラ状態だけリセットされ、命令は保持される
- コインを全取得でクリア表示
- 命令完走＆未クリアで「プログラムが実行されました」を表示
- マップ外に出たらリセット
