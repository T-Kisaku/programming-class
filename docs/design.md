# 設計方針

## 基本原則

### 1. 既存コードの尊重

- 純粋関数型のゲームロジックは変更せず、そのまま活用
- Zod スキーマによる型定義と検証を維持
- Svelte ストアパターンを継続して使用

### 2. 拡張性

- エディタ機能は既存のゲームロジックに依存するが、独立して開発可能
- 新しいレベル定義形式は既存のレベルスキーマと互換性を持つ

### 3. シンプルさ

- 追加機能は最小限の複雑さで実装
- 必要な機能だけを実装し、過度な抽象化を避ける

---

## 追加する機能

### 機能 1: レベルエディタ

#### 目的
学生が独自のステージを作成できるようにする

#### 実装要件

##### グリッドベースの編集UI
- クリックでタイルを選択・配置
- ドラッグで連続配置（オプション）
- 削除機能

##### タイル配置オプション
- **床 (floor)**: プレイヤーが移動できるタイル
- **壁 (wall)**: 移動をブロックするタイル
- **コイン (coin)**: タイルにコインを配置

##### タイル色の設定
- 定義されたカラーパレットから選択
- 色の追加・編集（オプション）

##### 開始位置の設定
- グリッド上で開始位置をクリックして設定
- 開始方向（N/E/S/W）を設定

#### UI コンポーネント

| コンポーネント | 責務 |
|--------------|------|
| `Toolbar.svelte` | タイル選択パレット、ツール切り替え |
| `GridEditor.svelte` | 編集可能なグリッド表示、クリックハンドリング |
| `LevelProperties.svelte` | レベルの基本設定（タイトル、サイズなど） |

---

### 機能 2: URL での共有

#### 目的
作成したレベルを簡単に他の学生に共有できるようにする

#### 実装要件

##### URL エンコード
- レベル定義を JSON 文字列化
- Base64 エンコードで URL に埋め込み
- 圧縮（オプション、必要に応じて）

##### URL デコード
- URL クエリパラメータからレベル定義を読み込み
- Base64 デコードと JSON パース
- Zod スキーマによる検証

##### 共有 UI
- 生成された URL を表示
- ワンクリックでコピー機能
- URL の短縮（オプション）

#### UI コンポーネント

| コンポーネント | 責務 |
|--------------|------|
| `ShareURL.svelte` | 共有URLの生成と表示、コピー機能 |

## データフロー

### レベル作成フロー

```
[ユーザー操作]
    ↓
[エディタストア] ← 更新
    ↓
[LevelDefinition] ← 生成
    ↓
[URLエンコード] ← 変換
    ↓
[共有URL] ← 表示/コピー
```

### レベル再生フロー

```
[URLクエリパラメータ]
    ↓
[URLデコード]
    ↓
[LevelDefinition] ← 検証
    ↓
[ゲームストア] ← 初期化
    ↓
[ゲームUI] ← 表示
```

---

## 制約と考慮事項

### URL 長さの制限

- URL クエリパラメータの長さにはブラウザやサーバーの制限がある
- 大きなレベルを作成した場合、URLが長くなりすぎる可能性がある
- 対応策：
  - レベルサイズの上限を設ける（例: 10x10）
  - 必要に応じて圧縮を導入
  - 将来的にはバックエンド保存+ID共有方式への移行

### セキュリティ

- URL からロードするレベルは Zod スキーマで厳密に検証
- 悪意のある入力に対して安全

### ユーザビリティ

- エディタは直感的に操作できる必要がある
- 不正な状態（例: 開始位置が壁の上）を防止する
- 適切なエラーメッセージを表示

## 将来の拡張可能性

### 拡張機能案

1. **プログラムエディタ**
   - プログラム（コマンド列）の編集機能
   - プログラムを含めてレベルを共有

2. **複数レベル管理**
   - ローカルストレージへの保存
   - 複数の自作レベルの管理

3. **インポート/エクスポート**
   - JSON ファイルへのエクスポート
   - JSON ファイルからのインポート

4. **レベルプレビュー**
   - 作成中のレベルで実際にプレイできる

5. **テンプレート**
   - 既存のレベルをテンプレートとして編集開始

## デザイン決定

### ストアの構成

- `editorStore.ts`: エディタ専用のストア
  - `gameState.ts` は維持
  - エディタストアは `LevelDefinition` を完全に管理
  - ゲームへの遷移時に `LevelDefinition` を `gameState` に渡す

### コンポーネントの階層

```
editor/+page.svelte
├── Toolbar.svelte
├── GridEditor.svelte
├── LevelProperties.svelte
└── ShareURL.svelte
```

### ステート管理

- Svelte の `writable` ストアを使用
- コンポーネント間でのデータ共有を実現
- リアクティブな更新を実現
